# Тестовое задание Avito

## Описание

Приложение представляет собой личный кабинет продавца с возможностью:

- просматривать список объявлений;
- открывать карточку объявления;
- редактировать объявление;
- получать AI-рекомендации для описания и цены.

## Стек

- React
- TypeScript
- Vite
- React Router
- TanStack Query
- React Hook Form
- Zod
- CSS Modules / CSS

## Реализованный функционал

### Страница списка объявлений `/ads`

- отображение списка объявлений;
- поиск по названию;
- фильтрация по категориям;
- фильтр "только требующие доработки";
- сортировка;
- пагинация по 10 элементов;
- переход на страницу объявления;
- переключение вида список/сетка.

### Страница объявления `/ads/:id`

- отображение полной информации по объявлению;
- блок "Требуются доработки";
- переход к редактированию;
- возврат к списку объявлений.

### Страница редактирования `/ads/:id/edit`

- форма редактирования;
- динамические поля в зависимости от категории;
- валидация формы;
- счётчик символов для описания;
- сохранение изменений через API;
- возврат без сохранения;
- сохранение черновика в `localStorage`;
- AI-генерация описания;
- AI-рекомендация цены.

## Что не реализовано

- unit-тесты;
- docker compose;
- отмена запросов при переходе между страницами.

## Установка и запуск

### Клонировать репозиторий

```bash
git clone <repo-url>
cd <repo-folder>
```

### Запуск Backend

```bash
cd server
npm install
$env:PORT=8081
npm start
```

### Запуск Frontend

```bash
cd client
npm install
npm run dev
```

### Ollama

### Установка

1. Установить Ollama
2. Скачать модель:

```bash
ollama pull llama3.1:8b
ollama serve
```

### Описание ИИ

В vite.config.js реализовано:

```bash
server: {
  proxy: {
    '/ollama': {
      target: 'http://127.0.0.1:11434',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/ollama/, ''),
    },
  },
},
```

Запрос выполняется:

```bash
fetch('/ollama/api/generate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    model: 'llama3.1:8b',
    prompt,
    stream: false,
  }),
});
```

Модель работает локально и не использует внешние API.

## Принятые решения

- **Частично поломанный сервер**
  Сервер не возвращал id объявлений и не был настроен на фильтр по цене.

- **Управление серверным состоянием**  
  Для работы с API выбран `TanStack Query`, так как приложение активно работает с загрузкой списков, детальной карточки и обновлением данных. Это позволило упростить работу с кэшированием, загрузками и ошибками.

- **Обновление данных после редактирования**  
  После успешного редактирования объявления используется инвалидирование кэша (`invalidateQueries`) и частично обновление данных (`setQueryData`).  
  Это позволяет отображать актуальные данные сразу после перехода на страницу объявления без ручной перезагрузки.

- **Работа с формой**  
  Для управления формой выбран `React Hook Form`, так как он обеспечивает минимальное количество перерендеров и удобную интеграцию с TypeScript.

- **Валидация данных**  
  Используется `Zod` для описания схем данных и валидации формы. Это обеспечивает единый источник правды для типов и проверок.

- **Динамические поля формы**  
  Поля характеристик рендерятся в зависимости от выбранной категории объявления (`auto`, `real_estate`, `electronics`).  
  Это упрощает масштабирование и делает форму более гибкой.

- **Сохранение черновика**  
  Данные формы сохраняются в `localStorage`, чтобы пользователь не терял введённую информацию при обновлении страницы или случайном уходе.

- **AI-интеграция**  
  Для генерации описания и рекомендации цены используется LLM через `Ollama`.  
  В модель передаётся контекст объявления (название, категория, характеристики), на основе которого формируется ответ.

- **Работа с изображениями**  
  Используются placeholder-изображения из `public`, так как загрузка реальных изображений не требовалась по условию задания.

- **Отказ от адаптива**
  Из-за нехватки времени пришлось отказаться от адаптива

- **Управление роутингом**  
  Используется `react-router-dom` с разделением на страницы `/ads`, `/ads/:id`, `/ads/:id/edit`, что соответствует требованиям задания.

- **Обработка состояний**  
  На всех страницах реализована обработка состояний загрузки (`loading`) и ошибок (`error`), чтобы обеспечить корректный пользовательский опыт.
